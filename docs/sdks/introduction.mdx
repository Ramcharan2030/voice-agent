SPX Voice ships official SDKs for **Python** and **TypeScript** that wrap the SPX Voice REST API and the workflow builder. Use them to create or edit agents, place outbound calls, and inspect runs from your own code.

- **Python** — [`spx-voice-sdk`](https://pypi.org/project/spx-voice-sdk/) on PyPI
- **TypeScript** — [`@spx-voice/sdk`](https://www.npmjs.com/package/@spx-voice/sdk) on npm

Both packages are generated from the same backend OpenAPI spec, so the method surface is equivalent — only the naming convention differs (`snake_case` in Python, `camelCase` in TypeScript).

## Install

<CodeGroup>
```bash Python
pip install spx-voice-sdk
```
```bash TypeScript
npm install @spx-voice/sdk
```
</CodeGroup>

## Authenticate

Generate an API key at [`/api-keys`](https://your-spx-voice-domain/api-keys) (or `http://localhost:3010/api-keys` for self-hosted). See [API Keys](/configurations/api-keys) for details.

Both SDKs read the API key from the `SPX_VOICE_API_KEY` environment variable by default, and the base URL from `SPX_VOICE_API_URL` (defaults to `http://localhost:8000`). You can also pass them explicitly.

<CodeGroup>
```python Python
from spx_voice_sdk import SPXVoiceClient

client = SPXVoiceClient(
    base_url="https://your-spx-voice-domain",
    api_key="YOUR_API_KEY",
)
```
```typescript TypeScript
import { SPXVoiceClient } from "@spx-voice/sdk";

const client = new SPXVoiceClient({
    baseUrl: "https://your-spx-voice-domain",
    apiKey: "YOUR_API_KEY",
});
```
</CodeGroup>

<Note>
For self-hosted deployments, swap `baseUrl` for your backend URL (e.g. `http://localhost:8000`).
</Note>

## Quick tour

List the agents in your workspace:

<CodeGroup>
```python Python
workflows = client.list_workflows()
for wf in workflows:
    print(wf.id, wf.name)
```
```typescript TypeScript
const workflows = await client.listWorkflows();
for (const wf of workflows) {
    console.log(wf.id, wf.name);
}
```
</CodeGroup>
