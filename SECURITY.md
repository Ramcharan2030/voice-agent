# Security Policy

## Supported Versions

Use the latest release or the latest `main` branch when possible. Security fixes
will be documented in release notes once the project has a formal release
process.

## Reporting a Vulnerability

Do not report vulnerabilities in public GitHub issues.

Use this repository's private vulnerability reporting channel once it is
enabled. Until then, contact the maintainers through the private security
contact listed in the repository metadata.

Please include:

- A short description of the vulnerability.
- Steps to reproduce it.
- The affected commit, branch, or deployment version.
- Any logs or screenshots that help explain the issue, with secrets removed.

We will acknowledge the report, investigate it, and coordinate a fix before
public disclosure.
